A functional CSS library built for speed and granular customization.

In Elassus, the modules are customizable in every way. Want more color options? Overwrite the scss variable that contains them with your own. Don't need a particular module? Don't include it in the build. Need a module of your own? Create one using the patterns laid out in the sample file. Don't like the class name syntax? Overwrite the function that builds them with your own preference.

Small

At it’s largest, Elassus is less than 15kb gzipped. That means your project isn’t going to turn in to a bloated mess of ever growing CSS.

Customizable

Other style libraries may claim it, but finding that to be untrue was the initial inspiration for creating Elassus. Finally, you can always get what you want.

Unbreakable

No more having to audit your project with every small change to your CSS to see if it broke something. In Elassus you change class names, not styles.

Understandable

Within minutes you’ll be reading and writing markup using Elassus. Imagine knowing the styles applied to your code without looking at the CSS.

Uniqueness

Finally, the freedom to create something that doesn’t look like a billion other products out that because it was built with a popular framework.

Responsive

The modules that most often need responsive breakpoints have them right out of the box. If you need more control, you can easily bend it to your will.

Getting Started

Elassus is really easy to get up and running in your project. If you want full control you can fork the repo or copy the files directly in to your scss directory, or use one of the following methods:

NPM

npm install elassus --save-dev

Bower

bower install elassus --save-dev

Module Class Names

A lot of time went in to determining the class name syntax to use for Elassus. We wanted to do our best to make sure that each selector was not only concise (As seen in libraries such as Tachyons and Basscss), but easy to read and understand what it was responsible for (Like you see in tools like Atomic CSS). With that being said, what follows is a map of the naming syntax developed:

<context>-<modifier>(<value>)<breakpoint>:<pseudo>
  • context: Calls out the primary declaration of the style (i.e., padding, text-align, justify-content, etc).
  • modifier: Identifies an optional subset of the primary declaration (e.g., top, bottom, color, etc)
  • value: The primary value being added to the declaration (e.g., 12col, center, gray, etc)
  • breakpoint: An optional breakpoint to limit the style to (e.g., small, medium, large, etc)
  • pseudo: An optional pseudo class to apply the styles to (e.g., focus, hover, active, etc)

With all this you are able to have classes like the following:

  • p-t(2)l: Adds padding-top with a value of 0.5rem (The second value in the size scale).
  • ta(c): Aligns text to the center.
  • bg-c(gray): Adds a background color of "gray"
  • w(50%): Sets the width to 50%
  • c(gray-dark):h: Sets the text color to "dark gray" on hover
